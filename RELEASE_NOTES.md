### KryoFlux floppy dumps
Disc Xplorer now opens **KryoFlux raw stream dumps** — point it at any `.raw` file in a dump folder and it decodes the flux, mounts the floppy, and lets you browse and extract files.

- **Flux-level decoding** — MFM bit-clock recovery with a PLL, IBM address/data-mark detection, and per-sector CRC verification; multiple captured revolutions act as free retries.
- **FAT12 / FAT16** — boot-sector geometry, long file names (VFAT), and file timestamps; 720 KB, 1.2 MB, and 1.44 MB layouts auto-detected.
- Works with any DTC file naming (`track00.0.raw` or `My Disk(1of1)00.0.raw`).

### Deleted-file recovery
Erased-but-recoverable directory entries are now listed with a **trashcan icon** and can be extracted:

- **FAT12/16 floppies** — `0xE5`-marked entries with the classic contiguous-cluster undelete heuristic (first name character is lost; shown as `_`).
- **FATX / XTAF** (Xbox) — the full name survives deletion and is recovered exactly.
- **UDF** (packet-written CD-RW/DVD±RW) — deleted file identifiers are flagged instead of being silently mixed into the listing.

### Smarter damaged-file detection
The red-X overlay now validates each sector's embedded **EDC checksum** in addition to the sync mark — catching sectors that were read with errors or regenerated by a dumper, not just missing ones. Works on both raw bin/cue and redumper `.scram` images, with no sidecar files needed.

### Other
- Damage flags no longer show false positives on HFS volumes of hybrid discs
- Sharper vector icons for damage/deletion markers, vertically centered
